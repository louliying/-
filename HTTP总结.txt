1. 3次握手：
	三次握手，即TCP连接的建立。
	TCP连接，必须是一方主动打开，另一方被动打开。
	a. 客户端向服务器端发送一段TCP报文
	b. 服务器端接收来自客户端的TCP报文后，结束LISTEN阶段，并返回一段TCP报文
	c. 客户端接收来自服务器端的确认收到数据的TCP报文后，明确了从客户端到服务器端的数据传输是正常的，结束SYN-SENT阶段，并返回最后一段TCP报文。
	d. 服务器端收到客户端的“确认收到服务器数据”的TCP报文后，明确服务器到客户端的数据传输是正常的。结束SYN-SENT阶段，进入ESTABLISHED阶段。
	e. 客户端和服务器端进行正常数据传输。


	简单来说：
	a. 客户端，带着报头，请求服务器端， “我要和你建立连接"
	b. 服务器端，收到报头后说， "好的，你可以传的"
	这是第一次
	c. 客户端接到服务器端的报头， 知道服务器端可以通信， 再传报头给服务器端，说”我收到你说的可以通信了"
	d. 服务器端收到客户端的确认收到的报头后， 改变状态，等客户端传数据了。
	这是第二次
	e. 客户端发数据请求
	f.服务器端相响请求，返回data
	这是第三次



	1991年： HTTP0.9
	1996:	 HTTP1.0
	1999:  	HTTP 1.1
	2015	HTTP 2.0

	一个TCP连接上，可以发送多个http请求，不同的HTTP版本，不一样。
	HTTP/1.0中， TCP连接与HTTP请求同步创建，http请求发至服务器，服务器响应后， 这个TCP链接就关闭了。

	HTTP/1.1 中，TCP链接可以一直保持， 一个http请求完后，另一个请求可以接碰着传输。 就没有三次握手的消耗

	HTTP/2 中， 同一个TCP连接可以并发传输http请求


2. 输入一个url，经历了什么？
	1). 解析URL是否合法
	2). 浏览器检查是否有缓存：浏览器，系统，路由的缓存
		如果有缓存， 直接显示，
		没有缓存，跳入步骤c
	3). 解析域名（DNS解析），解析获取对应的IP地址， 发送http请求
	4). 浏览器向服务器发起TCP链接，三次握手
	5). 握手成功后，浏览器向服务器发送http请求，请求数据包
	6). 服务器收到请求后，处理后，将数据返给浏览器
	7). 浏览器收到http响应
	8). 浏览器解析响应，如以缓存，则将响应缓存
	9). 浏览器发送嵌入HTML中的静态资源的请求
	10). 如有异步请求，发送异步请求
	11). 页面渲染结束 

3. get post的区别
	Get: 浏览器会把http header 和 data一并发送出去，服务器响应200，返回数据

	Post: 浏览器先发送header，服务器响应100 continue后，浏览器再发送data, 服务器响应200 ，并返回数据 

	所以， get一个TCP数据包， post二个TCP数据包

4. HTTP和HTTPS的区别：
	https: http + ssl
	1). https 有ca证书， http没有
	2). http是超文本传输协议，信息是明文传输， https是具有安全性的ssl加密传输协议
	3). http默认端口80， https默认443
